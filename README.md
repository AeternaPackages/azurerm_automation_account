## Authoring convention

This package exposes a single input variable, `automation_accounts`, keyed by the azurerm_automation_account map
key you choose. Related child resources nest inside each entry (see `variables.tf`); nested
map keys are also your choice.

- Ancestor relationships are automatic: an FK field pointing at a direct ancestor (e.g. the
  account name on a nested schedule) does not appear in the type at all - it is always filled in
  from the real created resource, using the nesting position alone.
- Map keys must not contain `/`. Keys are joined with `/` internally to build a globally
  unique key per nested resource, so a `/` inside one of your keys can silently merge two
  different entries into one. A `validation` block rejects this at plan time, but it is
  simplest to just avoid `/` in keys.
- `azurerm_automation_job_schedule`.`runbook_name`: set to the map key you used for the `azurerm_automation_runbook` entry elsewhere in this package to auto-wire it, or to a literal external value otherwise.

A lookup that does not match any key resolves to the literal value you supplied instead of
failing at plan time - useful for referencing something outside this package, but also means a
typo in a sideways key silently becomes a plain (probably wrong) string rather than an error.
